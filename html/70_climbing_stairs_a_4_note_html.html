LeetCode 70. 爬楼梯 —— 解题思想总结

本题要求计算到达第 n 级台阶的不同方法数，每次可走 1 步或 2 步。本质是经典的斐波那契数列建模问题，可通过多种方法求解，逐步优化效率。

↻V1 - 暴力递归（未优化）
思想：

到达第 n 级的方法 = 到达第 n-1 级的方法 + 到达第 n-2 级的方法

即：climbStairs(n) = climbStairs(n-1) + climbStairs(n-2)

实现： 
if n <= 2: return n
return self.climbStairs(n-1) + self.climbStairs(n-2)
缺点：
  • 存在大量重复子问题（如多次计算 climbStairs(3)）
  • 时间复杂度：O(2^n)（指数级），不可接受
  • 空间复杂度：O(n)（递归栈深度）
📌 仅用于理解递归结构，实际不可用
🧠V2 - 记忆化递归（自顶向下 DP）
思想：

在递归基础上引入缓存数组 dp 或哈希表，保存已计算结果，避免重复调用。

关键点：
  • 使用长度为 n+1 的数组，使得 dp[i] 对应第 i 级台阶的方案数
  • 先查缓存，命中则直接返回；否则递归计算并存储
复杂度：
  • 时间复杂度：O(n)（每个状态只计算一次）
  • 空间复杂度：O(n)（递归栈 + 缓存数组）
✅ 优点：逻辑清晰，易于从暴力递归转化
💡 是动态规划的入门范式之一
▥V3 - 动态规划（自底向上，数组实现）
思想：
  • 状态定义：dp[i] 表示爬到第 i 级的方法总数
  • 转移方程：dp[i] = dp[i-1] + dp[i-2]
  • 初始条件：dp[0] = 1, dp[1] = 1
遍历顺序：

从小到大递推至 n

复杂度：
  • 时间：O(n)
  • 空间：O(n)
📌 适合展示完整 DP 思维流程
🚀V4 - 滚动数组优化（最优解）
思想：

观察发现：当前状态只依赖前两个状态 → 不需要整个数组，只需两个变量滚动更新。

实现： 
a, b = 1, 1
for _ in range(2, n+1):
    a, b = b, a + b
return b
复杂度：
  • 时间复杂度：O(n)
  • 空间复杂度：O(1) ✅ 最优
📌 推荐作为最终提交版本，体现空间优化意识

🔍 四种方法对比总结

方法 时间复杂度 空间复杂度 是否推荐 说明
V1 暴力递归 O(2^n) O(n) ❌ 否 效率极低
V2 记忆化递归 O(n) O(n) ⭕ 可接受 展示 DP 思维
V3 动态规划数组 O(n) O(n) ✅ 是 清晰易懂
V4 滚动数组 O(n) O(1) ✅✅ 强烈推荐 工业级最优解

✅ 一句话总结：

本题从“斐波那契”关系出发，通过记忆化递归 → 动态规划 → 滚动数组优化，实现了从指数时间到线性时间再到常数空间的演进过程，是学习动态规划设计与优化技巧的经典范例。
