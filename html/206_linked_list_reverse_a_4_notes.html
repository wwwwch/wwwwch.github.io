206. 反转链表 —— 解题思想总结

本题要求将单向链表原地反转，常见解法可分为三类：递归回溯法、显式状态递归法、双指针迭代法。以下是各方法的核心思想与对比。

回溯反转递归到尾，再反向改指针
状态传递pre / cur 作为递归状态
双指针迭代空间 O(1)，首选写法
1

V1 - 递归法（自底向上回溯反转）

思想：利用函数调用栈“递归到末尾，再逐层返回”的特性，在回溯过程中完成指针反转。

关键操作：

new_head = self.reverseList(head.next)  # 递归得到新头
head.next.next = head                   # 让后继节点指回头部
head.next = None                        # 断开原连接，防止环

特点：

  • 简洁优雅，逻辑集中于回溯阶段。
  • 依赖系统栈，空间复杂度为 O(N)。
  • 时间复杂度：O(N)
📌 适用场景：理解递归本质，适合教学演示。
2

V1.1 - 递归法（显式传参模拟尾递归）

思想：显式传递当前节点 cur 和前驱节点 pre，模拟迭代中的双指针行为，把状态通过参数传递。

关键操作：

def recur(self, pre: ListNode, cur: ListNode) -> ListNode:
    """
    递归反转链表的辅助函数
    pre: 当前节点的前驱节点（反转后的 next 指向）
    cur: 当前节点
    return: 新链表的头节点（原链表尾节点）
    """
    if not cur:
        return pre  # 终止条件：到达原链表末尾，pre 即为新头

    head = self.recur(cur, cur.next)    # 递归处理后续节点

    cur.next = pre  # 将当前节点的 next 指向前驱节点，完成反转

    return head     # 返回最终的新头节点（始终是最后一个返回的 pre）

特点：

  • 更接近迭代逻辑，结构清晰。
  • 虽仍使用递归栈，但状态管理更明确。
  • 时间复杂度：O(N)，空间复杂度：O(N)
📌 优势：易于从迭代版本转换而来，便于理解“状态传递”。
3

V2 - 双指针迭代法

思想：使用两个指针 prev 和 curr，从前向后遍历链表，逐步修改每个节点的 next 指针。

关键步骤：

  1. 保存 next_node = curr.next
  2. 将 curr.next = prev
  3. prev = curr，curr = next_node

特点：

  • 原地操作，无需额外数据结构。
  • 空间复杂度仅 O(1)
  • 时间复杂度：O(N)
  • 工业级推荐写法
📌 一句话口诀：“暂存下一，反向链接，双指针前进。”

🔍 三种方法对比总结

方法 时间复杂度 空间复杂度 是否原地 思维难度 推荐程度
V1 递归回溯 O(N) O(N) 是 中等 ⭐⭐⭐☆
V1.1 显式递归 O(N) O(N) 是 中 ⭐⭐⭐⭐
V2 双指针迭代 O(N) O(1) 是 低 ⭐⭐⭐⭐⭐
✅ 总体总结一句话：递归法重在理解“回溯”与“状态传递”，而双指针迭代法以最优空间效率成为首选实现方式。掌握这三种方法，可全面应对链表反转类问题。
