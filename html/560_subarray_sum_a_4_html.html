560. 和为K的子数组 —— 前缀和+哈希表

🎯 问题本质
统计数组中和为 k 的连续子数组的个数。
📊 方法对比
方法 核心思路 时间复杂度 空间复杂度 特点
暴力枚举 固定起点，累加终点 O(n²) O(1) 直观但慢
前缀和+哈希表 prefix[j] - prefix[i] = k → 查找 prefix[i] = prefix[j] - k 出现次数 O(n) O(n) 最优解
🔍 方法详解
1暴力枚举（O(n²)）
思路：固定每个起点 i，累加 i 到 j 的和，等于 k 时计数。 
def subarraySum(self, nums, k):
    total = 0
    n = len(nums)
    for i in range(n):
        s = 0
        for j in range(i, n):
            s += nums[j]
            if s == k:
                total += 1
    return total
2前缀和+哈希表（O(n)）
核心转化：
子数组和 = prefix[j] - prefix[i] = k
⇒ prefix[i] = prefix[j] - k
遍历时，对于当前前缀和 cur_sum，查找哈希表中 cur_sum - k 的出现次数，即为以当前元素结尾且和为 k 的子数组个数。
  • 用哈希表存储前缀和 → 出现次数
  • 初始化 prefix_count[0] = 1（空子数组）
def subarraySum(self, nums, k):
    count = 0
    cur_sum = 0
    prefix_count = defaultdict(int)
    prefix_count[0] = 1

    for num in nums:
        cur_sum += num
        count += prefix_count[cur_sum - k]   # 查找能凑成k的前缀和
        prefix_count[cur_sum] += 1          # 记录当前前缀和

    return count
⚠️ 关键细节
细节 说明
为什么要存前缀和出现次数 可能有多个不同的起始位置得到相同的前缀和
初始化 prefix_count[0]=1 处理从开头到当前位置的和正好等于 k 的情况
遍历时先查后存 避免当前前缀和自己被用于 cur_sum - k = 0 的情况（子数组需非空）
适用性 数组元素可为负数，哈希表法依然有效
💎
一句话总结
前缀和转化子数组和为两数之差，哈希表记录前缀和出现次数，一次遍历 O(n) 统计所有满足条件的子数组个数。
