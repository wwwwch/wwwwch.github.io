📚 347.前K个高频元素 - 解法总结

哈希表 · 排序 · 桶排序

1🎯 题目本质

给定整数数组 nums 和整数 k，返回出现频率最高的前 k 个元素。

答案顺序任意，核心是：先统计频率，再选出最高的 k 个。

2📊 两种方法对比

方法 核心思想 时间 空间 结构 特点
字典排序法 统计频率后对键按值降序排序 O(n log n) O(n) 哈希表 + 排序 简单，适合小数据
桶排序法 利用频率范围有限，建立频率桶 O(n) O(n) 哈希表 + 列表数组 线性时间，适合大数据

3.1字典排序法

核心步骤

  1. 用哈希表 count 统计每个数字的出现频率。
  2. 对哈希表的键按照频率值降序排序。
  3. 返回排序后前 k 个键。
from collections import defaultdict
count = defaultdict(int)
for num in nums:
    count[num] += 1

# 按频率降序排序键，取前k个
result = sorted(count.keys(), key=lambda x: count[x], reverse=True)
return result[:k]

关键点

  • 排序对象是 键，排序依据是 值。
  • 瓶颈在排序，最坏时间复杂度 O(n log n)。

3.2桶排序法

核心思想

频率范围是 [1, n]，创建 n+1 个桶：下标 i 的桶存放所有频率为 i 的数字；再从高频到低频收集结果。

实现步骤

  1. 统计频率：count 字典（数字 → 频率）。
  2. 创建桶：
buckets = [[] for _ in range(len(nums) + 1)]

索引 freq 表示频率，buckets[freq] 存放该频率的数字。

  1. 将每个数字放入对应频率的桶中：
for num, freq in count.items():
    buckets[freq].append(num)
  1. 从最大频率 n 向下遍历，直到收集 k 个：
result = []
for freq in range(len(nums), 0, -1):
    for num in buckets[freq]:
        result.append(num)
        if len(result) == k:
            return result

关键点

  • 桶数量为 n+1：频率最大为 n，频率 0 不使用。
  • 从高频向低频遍历，保证优先取高频元素。
  • 总时间复杂度 O(n)：统计 O(n)，装桶 O(m)，收集 O(n)。

4⚠️ 注意事项

  • 频率统计：两种方法都依赖字典记录频率。
  • 桶排序前提：频率范围有限且已知（1~n）。
  • 结果顺序：题目不要求顺序；桶排序天然按频率从高到低收集。

5📊 复杂度分析对比

维度 字典排序法 桶排序法
时间复杂度 O(n log n) O(n)
空间复杂度 O(n) O(n)
适用场景 数据较小或频率分布较散 大规模数据，追求线性时间

6💡 选择建议

小数据：字典排序更直接
大数据：桶排序更优
本质：以空间换时间
  • 如果数据量不大（例如 n < 105），两种方法都可，字典排序更简单。
  • 如果追求最优时间复杂度，或 n 非常大，桶排序是更好的选择。
  • 桶排序利用频率作为下标，避免了排序。
💎 一句话总结：前K高频元素有两种经典解法：字典排序法直观但需 O(n log n)；桶排序法利用频率范围构建桶，达到 O(n) 线性时间，是本题最优解。
