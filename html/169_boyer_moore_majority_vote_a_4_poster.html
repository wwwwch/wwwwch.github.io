169. 多数元素 - Boyer-Moore投票算法

O(n) 时间 · O(1) 空间
🎯 核心问题

找出出现次数超过 n/2 的元素（保证一定存在）。

💎 一句话总结

多数元素超过半数，一对一对不同数相互抵消，最后剩下的必是多数元素。

🔍 Boyer-Moore投票算法思想
洞察点 数学原理 通俗理解
计数抵消 多数元素 > n/2，其它元素总和 < n/2 每对不同的数互相抵消，剩下的必是多数元素
等效删除 删除两个不同的数，多数元素仍占多数 一对一抵消，多数元素总能存活到最后
🎮 算法模拟示例
数组：[2, 2, 1, 1, 1, 2, 2]
num=2: count=0 → major=2, count=1✅ 候选2
num=2: major相同 → count=2✅ 支持
num=1: major不同 → count=1⚡ 抵消
num=1: major不同 → count=0⚡ 完全抵消
num=1: count=0 → major=1, count=1🔄 候选1上台
num=2: major不同 → count=0⚡ 抵消
num=2: count=0 → major=2, count=1🔄 候选2上台
最终：major = 2 ✅
🔬 算法正确性证明
数学原理：假设多数元素为 m，出现次数为 k > n/2；其它元素总数为 n-k < n/2。每次抵消删除一个 m 和一个非 m，最终至少剩余 k - (n-k) = 2k-n > 0 个 m。

情况1：候选是多数元素

🟢🟢🟢🟢 ⚪⚪⚪
多数元素支持自己，其它元素反对；由于多数元素超过一半，最终 count > 0。

情况2：候选不是多数元素

🔴🔴 🟢🟢🟢🟢
多数元素全体反对此候选，即使部分其它元素支持，count 也会归零，候选下台。
⚙️ 算法实现逻辑：三行核心逻辑
major = 0     # 当前候选多数元素
count = 0     # 候选元素的净支持票数

for num in nums:
    if count == 0:    # 1. 当前无候选 → 新候选上台
        major = num
    if num == major:  # 2. 相同元素 → 支持票+1
        count += 1
    else:             # 3. 不同元素 → 反对票-1（抵消）
        count -= 1
🔎 关键步骤解析
步骤 条件 操作 含义
1 count == 0 major = num 当前没有占优候选，新元素成为候选
2 num == major count += 1 遇到相同元素，支持候选
3 num != major count -= 1 遇到不同元素，抵消一票
📊 复杂度分析
维度 性能 说明
时间复杂度 O(n) 只需一次线性扫描
空间复杂度 O(1) 仅用两个变量
比较次数 n次 每个元素比较一次
💡 与其他方法对比
方法 时间复杂度 空间复杂度 特点
哈希表统计 O(n) O(n) 直观但需要额外空间
排序取中位数 O(n log n) O(1) 排序破坏原数组
分治法 O(n log n) O(log n) 递归开销
Boyer-Moore O(n) O(1) ✅ 最优解

⚠️ 使用前提

必须保证多数元素存在（出现次数 > n/2）。如不确定存在，需要二次验证遍历计数。

💎 结论

Boyer-Moore 通过“不同元素相互抵消”保留多数元素，时间复杂度 O(n)，空间复杂度 O(1)，是该问题的最优解法。
