📚 283.移动零 - 双指针解法总结

1 🎯 核心问题
将数组中的零移动到末尾，同时保持非零元素的相对顺序。
2 📊 两种方法对比
方法 思路 时间复杂度 空间复杂度 特点
双指针覆盖法 两次遍历：先覆盖非零元素，后补零 O(2n) → O(n) O(1) 逻辑直观，操作简单
双指针交换法 一次遍历：遇非零元素时交换到前面 O(n) O(1) 效率更高，一次完成
3 🔍 方法详解
1 双指针覆盖法（两次遍历）

核心思想：先把所有非零元素移动到前面，然后在后面补零。

cur = 0
# 第一次遍历：将非零元素覆盖到前面
for k in range(n):
    if nums[k] != 0:
        nums[cur] = nums[k]
        cur += 1

# 第二次遍历：从cur开始填充0
for i in range(cur, n):
    nums[i] = 0
关键点：
  • cur 记录下一个非零元素应放置的位置
  • 第一遍完成后，cur 表示非零元素的数量
  • 优点：逻辑清晰，不会改变非零元素顺序
  • 缺点：需要两次遍历
2 双指针交换法（一次遍历）

核心思想：使用双指针，遇到非零元素时交换到前面位置。

cur = 0  # cur指向下一个非零元素应该放置的位置
for k in range(n):
    if nums[k] != 0:
        # 交换当前元素到cur位置
        nums[cur], nums[k] = nums[k], nums[cur]
        cur += 1

优化版本（避免自交换）：

cur = 0
for k in range(n):
    if nums[k] != 0:
        if k > cur:  # 避免自交换，提高效率
            nums[cur], nums[k] = nums[k], nums[cur]
        cur += 1
4 🔬 算法原理深度分析

✅ 双指针交换法的严谨证明

指针定义与维护的不变量

  1. cur 指针：指向下一个非零元素应该放置的位置
  2. 循环不变量：cur 之前的所有元素都是非零的，且保持了原数组的相对顺序

关键性质证明

定理：当 k > cur 时，nums[cur] 一定是 0。
证明：
  1. 初始化时，cur = 0，不变量成立（空数组）
  2. 假设在处理到某个位置 k 之前，不变量成立
  3. 当遇到 nums[k] ≠ 0 时：
    • 如果 k = cur：当前位置已经是正确位置，直接 cur += 1
    • 如果 k > cur：根据不变量，nums[cur] 是下一个非零元素放置位置；由于还未在此处放置非零元素，且 cur < k，所以 nums[cur] 一定是 0
  4. 交换后，nums[cur] 变为非零元素，不变量仍然成立

顺序保持的数学证明

  1. 非零元素相对顺序不变：算法按顺序遍历，先遇到的非零元素先放到前面，因此相对顺序保持不变。
  2. 零元素全部后移：每个非零元素都会前移到 cur 位置，cur 单调递增覆盖前部位置，零元素自然被交换到后部。
5 📊 复杂度对比
维度 覆盖法（两次遍历） 交换法（一次遍历）
时间复杂度 O(2n) → O(n) O(n)
空间复杂度 O(1) O(1)
赋值操作 n次覆盖 + (n-cur)次补零 最多cur次交换
交换次数 0 ≤非零元素个数
实际效率 两次遍历，但操作简单 一次遍历，条件判断多
💎 一句话总结

双指针法解决移动零问题：覆盖法两次遍历逻辑清晰，交换法一次遍历效率更优，两种方法都能保持非零元素顺序，根据场景选择合适方法。
