21. 合并两个有序链表

目标：把两个升序链表合并为一个升序链表，复用原节点。

链表双指针递归LeetCode 21

图解核心

1→2→4→ null
1→3→4→ null
合并⇒1→1→2→3→4→4
记忆：谁小接谁，指针后移；一边空了，直接接另一边剩余部分。

方法一：迭代（尾插法）

  1. 建 dummy 哨兵，cur 指向结果链表尾部。
  2. 比较 list1.val 与 list2.val，小者接到 cur.next。
  3. 被接入链表指针后移，cur 后移。
  4. 循环结束后，拼接剩余链表：cur.next = list1 or list2。
返回：dummy.next，不是 dummy。
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def mergeTwoLists(self, list1: Optional[ListNode], list2: Optional[ListNode]) -> Optional[ListNode]:
        cur = dummy = ListNode()    # 用哨兵节点简化代码逻辑
        while list1 and list2:
            if list1.val < list2.val:
                cur.next = list1    # 把 list1 加到新链表中
                list1 = list1.next
            else:
                cur.next = list2    # 把 list2 加到新链表中
                list2 = list2.next
            cur = cur.next
        # 拼接剩余链表
        cur.next = list1 or list2
        return dummy.next

方法二：递归（头插法）

  1. 终止：list1 空返回 list2；list2 空返回 list1。
  2. 选择头：较小节点作为当前合并链表的头。
  3. 递归接尾：头节点的 next 指向剩余链表的合并结果。
递归式：小头.next = merge(小头.next, 另一条链)
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def mergeTwoLists(self, list1: Optional[ListNode], list2: Optional[ListNode]) -> Optional[ListNode]:
        if list1 is None:
            return list2
        if list2 is None:
            return list1
        
        if list1.val < list2.val:
            list1.next = self.mergeTwoLists(list1.next, list2)
            return list1
        else:
            list2.next = self.mergeTwoLists(list1, list2.next)
            return list2

复杂度 & 对比

版本时间空间特点
V1 迭代O(m+n)O(1)最稳；面试首选
V2 递归O(m+n)O(m+n)代码短；有递归栈
迭代关键dummy + cur
尾部持续接小节点。
递归关键先定头，再接尾
返回当前较小节点。

易错点

  • 忘记移动：接入节点后，要移动对应的 list1/list2。
  • 忘记移动 cur：否则结果链表无法继续增长。
  • 返回错节点：迭代返回 dummy.next。
  • 漏接剩余：循环后必须 cur.next = list1 or list2。
  • 递归栈风险：链表很长时递归可能栈溢出。

速记卡

1比大小
2接小的
3接剩余
结论：优先写 迭代尾插法；递归写法用于理解“当前最小头 + 剩余递归合并”。
