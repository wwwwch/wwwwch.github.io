15.三数之和 - 排序+双指针解法总结

🎯 核心思路一览 排序定序 · 固定一数 · 双指针夹逼
步骤核心操作目的与作用
1. 排序 nums.sort() ✅ 为双指针法建立有序基础
✅ 使相同数字相邻，便于去重
2. 固定数 for k in range(n-2) 🎯 确定三元组的第一个元素
3. 双指针 i=k+1, j=n-1 while i<j 🔍 在剩余区间内搜索另外两个数
4. 移动策略 根据 total 值移动指针 🎮 total<0→i++　total>0→j--　total=0→记录并继续
⚠️ 三层去重机制 去重不是提前乱跳；是在正确时机跳过相邻重复值
去重位置代码逻辑作用时机
📍 固定数去重 if k>0 and nums[k]==nums[k-1]: continue 外层循环开始
📍 左指针去重 while i<j and nums[i]==nums[i+1]: i+=1 找到解后，记录前
📍 右指针去重 while i<j and nums[j]==nums[j-1]: j-=1 找到解后，记录前
🔧 优化策略对比 利用排序后的最小值 / 最大值判断是否还有搜索必要
优化类型判断条件效果
提前终止 if nums[k] > 0: break 第一个数>0，三数和必为正
最小和剪枝 nums[k]+nums[k+1]+nums[k+2] > 0 当前k的最小组合已>0
最大和剪枝 nums[k]+nums[-2]+nums[-1] < 0 当前k的最大组合仍<0
🎮 双指针移动规则 小了右移左指针 · 大了左移右指针
情况操作逻辑分析
total < 0 i += 1 和太小 → 需要增大左值
total > 0 j -= 1 和太大 → 需要减小右值
total = 0 1. 记录结果
2. i+=1; j-=1		 ✅ 找到有效解
✅ 继续寻找其他组合
📊 复杂度分析表 排序成本被主过程 O(n²) 主导
维度复杂度说明
时间复杂度 O(n²) 外层循环O(n) × 双指针O(n)
空间复杂度 O(1) 仅使用常数额外空间（除输出外）
排序开销 O(n log n) 预处理成本，被O(n²)主导
💡 关键洞察与启发 记住模式，比记单题更重要
🔍 问题转换思维
  • 无序 → 有序：排序改变了问题的搜索方式
  • 三重循环 → 双重循环：利用有序性将O(n³)降为O(n²)
  • 暴力枚举 → 智能搜索：双指针实现高效夹逼
⚠️ 易错点提醒
易错点正确做法错误后果
去重时机 找到解后立即去重 可能漏解或产生重复
指针移动 total=0时同时移动i和j 可能陷入死循环
边界处理 数组长度<3直接返回 无意义计算
🎯 模式应用

这种 “排序+固定+双指针” 模式适用于：

  • ✅ 三数之和/四数之和系列问题
  • ✅ 有序数组多目标搜索
  • ✅ 需要避免重复组合的场景
📌 一句话总结

排序定序，固定一数，双指针夹逼寻解；三层去重保唯一，三重优化提效率。
