📚 283. 移动零｜双指针速记

1核心任务

把 0 移到数组末尾；非零元素相对顺序不变；要求原地修改。

原地 O(1) 稳定顺序 双指针 一次/两次遍历

2图解思路

① 覆盖法：先非零，后补零
扫描 nums[k]
→
非零写到 cur
→
cur 后补 0

两次遍历；逻辑最直观。

② 交换法：遇非零就前放
扫描 nums[k]
→
非零与 cur 交换
→
cur++

一次遍历；可用 k > cur 避免自交换。

3方法对比

方法 核心动作 复杂度 记忆点
覆盖法 非零元素依次覆盖前部，再把尾部填 0 O(n) / O(1) 两段式
交换法 非零元素与 cur 位置交换 O(n) / O(1) 一次完成

4代码：覆盖法

cur：下一个非零元素应放置的位置；第一遍后，cur = 非零元素数量。

cur = 0
# 第一次遍历：将非零元素覆盖到前面
for k in range(n):
    if nums[k] != 0:
        nums[cur] = nums[k]
        cur += 1

# 第二次遍历：从cur开始填充0
for i in range(cur, n):
    nums[i] = 0
  • 优点：清晰、稳定、不改变非零顺序。
  • 缺点：两次遍历；写入次数可能更多。

5代码：交换法

cur：下一个非零元素应放置的位置；遇非零就交换到前面。

cur = 0  # cur指向下一个非零元素应该放置的位置
for k in range(n):
    if nums[k] != 0:
        # 交换当前元素到cur位置
        nums[cur], nums[k] = nums[k], nums[cur]
        cur += 1

优化：避免自交换

cur = 0
for k in range(n):
    if nums[k] != 0:
        if k > cur:  # 避免自交换，提高效率
            nums[cur], nums[k] = nums[k], nums[cur]
        cur += 1
易错点：cur += 1 必须在遇到非零后执行；不要放到交换条件 k > cur 里面。

6证明速记

1
不变量：cur 前都是非零，且顺序保持。
2
为什么 k > cur 时 nums[cur] 是 0：cur 是下一个非零放置位；若它已经是非零，早应被归入前缀。
3
为什么顺序不变：按 k 从左到右扫描，先遇到的非零先放进前缀。
4
为什么 0 到后面：非零持续填满前部，0 被覆盖或交换到后部区域。

7复杂度与选择

维度 覆盖法 交换法
时间O(2n) → O(n)O(n)
空间O(1)O(1)
操作覆盖 + 补零最多非零数次交换
适合讲解 / 初学面试 / 一次遍历
💎 一句话：用 cur 维护“非零前缀末尾”；覆盖法两步清晰，交换法一步更紧凑；核心是不变量：cur 前非零且有序。
