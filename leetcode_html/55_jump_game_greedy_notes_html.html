55. 跳跃游戏 —— 贪心解法

Jump Game · Greedy
1问题本质

给定非负整数数组 nums，初始位于下标 0，每个元素表示在该位置能跳跃的最大长度。判断能否到达最后一个下标。

2核心思路：贪心维护最远可达位置
  • 一次遍历，维护当前能到达的最远下标 max_right
  • 若当前位置 i 在可达范围内（i ≤ max_right），则更新 max_right = max(max_right, i + nums[i])
  • 若 max_right ≥ n-1，直接返回 True
  • 若当前位置 i > max_right，说明中间断开，无法继续，返回 False
3关键代码
def canJump(self, nums: List[int]) -> bool:
    n = len(nums)
    max_right = 0                     # 当前能到达的最远下标

    for i in range(n):
        if i > max_right:             # 当前位置不可达
            return False
        max_right = max(max_right, i + nums[i])
        if max_right >= n - 1:        # 已能到达终点
            return True

    return True
4复杂度分析
维度性能
时间复杂度O(n)（一次遍历）
空间复杂度O(1)（常数变量）
5关键细节
细节说明
可达性判断遍历时若 i > max_right，则无法到达当前位置，直接失败
更新时机每步都尝试扩展最远距离
提前终止一旦 max_right ≥ n-1 即可返回 True
无需 DP贪心即可，无需记录每个位置是否可达
6对比其他思路
方法 时间复杂度 空间复杂度 特点
贪心（本题）O(n)O(1)最优，只需维护最远距离
动态规划O(n²)O(n)可解但效率低
反向贪心O(n)O(1)从后往前找能到终点的最左位置
7一句话总结
!
维护当前能到达的最远下标；若遍历过程中当前位置不可达则失败，否则不断扩展最远距离直到覆盖终点。
