300. 最长递增子序列 LIS

目标：给定整数数组 nums，求其中严格递增的最长子序列长度。

📌 子序列：不要求连续 ⬆️ 严格递增：后者 > 前者 🎯 只求长度

核心记忆图

🧩
DP：“我能接在谁后面？”遍历前面所有更小元素。
🚀
贪心 + 二分：“同样长度，结尾越小越好。”维护每个长度的最小结尾。
进化链：DP 发现结构 → 贪心提炼状态 → 二分加速查找。

V1：动态规划｜O(n²)

适合小规模数据、面试展示 DP 思维。

状态定义

dp[i] = 以 nums[i] 结尾的最长递增子序列长度

转移公式

若 j < i 且 nums[j] < nums[i]：dp[i] = max(dp[i], dp[j] + 1)

步骤

  1. 初始化：所有 dp[i] = 1。
  2. 对每个 i，枚举前面所有 j。
  3. 若 nums[j] < nums[i]，尝试接上。
  4. 答案：max(dp)。
比喻：每个人问：“前面谁比我小？谁能带我走得最远？”
时间空间关键词
O(n²)O(n)枚举前驱、以 i 结尾

V2：贪心 + 二分｜O(n log n)

tails[L-1] 表示：长度为 L 的递增子序列中，最小结尾值。

核心：不关心具体路径，只关心每种长度下的“最优结尾”。结尾越小，未来越容易扩展。

更新规则

  • x > tails[-1]：直接追加，长度 +1。
  • 否则：找第一个 >= x 的位置并替换。
  • 查找用二分：因为 tails 始终有序。
性质结论原因
tails 有序✅追加或替换都保持递增
替换安全✅只优化结尾，不减少长度
可二分✅有序 → lower_bound
比喻：tails 是“潜力排行榜”：记录每种长度下最有前途的结尾。

对比速记

DP

  • 状态真实：每个 dp[i] 对应真实结尾。
  • 思路直观：枚举所有前驱。
  • 瓶颈：双循环。

Greedy + Binary

  • tails 不一定是一条真实 LIS。
  • 它是辅助状态表：维护最小结尾。
  • 用二分把查找降到 log n。
易错点：tails 的内容不是最终 LIS 本身；长度才是答案。
代码 1：动态规划 O(n²) 
class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        n = len(nums)
        # dp[i] 表示以 nums[i] 结尾的最长递增子序列长度
        dp = [1] * n

        for i in range(n):
            # 检查前面所有比 nums[i] 小的元素
            for j in range(i):
                if nums[j] < nums[i]:
                    # 可接在 nums[j] 后，更新长度
                    dp[i] = max(dp[i], dp[j] + 1)

        # 返回所有可能结尾中的最大长度
        return max(dp)
代码 2：贪心 + 二分 O(n log n) 
class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        tails = []  # 维护一个数组：tails[i] 表示长度为 i+1 的递增子序列的最小结尾值

        # 遍历每个数字，动态更新 tails 数组
        for i in range(len(nums)):
            tails = self.tails_update(nums[i], tails)

        # 最终 tails 的长度就是最长递增子序列的长度
        return len(tails)

    def tails_update(self, num_i: int, tails: List[int]) -> List[int]:
        """
        更新 tails 数组：
        - 若 num_i 大于所有结尾，则追加（延长序列）
        - 否则，替换第一个 >= num_i 的元素（保持相同长度下的最小结尾）
        """
        left, right = 0, len(tails)  # 注意：right 指向插入位置上限，不一定是最后一个索引

        # 使用二分查找找到第一个 >= num_i 的位置（即 bisect_left）
        while left < right:
            mid = (left + right) // 2
            if tails[mid] < num_i:
                left = mid + 1  # 往右找
            else:
                right = mid     # 往左找第一个满足的位置

        # 此时 left 就是要插入/替换的位置
        if left == len(tails):
            tails.append(num_i)      # 可以延长最长序列
        else:
            tails[left] = num_i      # 替换，维持更优结尾

        return tails

最后背诵版

LIS 求严格递增子序列最大长度。

DP：dp[i] 表示以 nums[i] 结尾，枚举 j < i 且 nums[j] < nums[i]。

优化：用 tails 记录每种长度的最小结尾；追加或替换第一个 >= x 的位置。

结论：len(tails) 就是答案；tails 本身不一定是最终序列。
