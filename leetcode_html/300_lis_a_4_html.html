300. 最长递增子序列 （LIS）

从 O(n²) 动态规划，到 O(n log n) 贪心 + 二分优化

🎯 题目核心要求

给定整数数组 nums，找出其中严格递增的最长子序列长度。

  • 子序列：不要求连续，但必须保持原顺序
  • 严格递增：后一个元素 > 前一个元素
  • 目标：只求最大长度，不要求输出具体序列

💡 核心进化链

DP 发现结构 → 贪心提炼状态 → 二分加速查找

状态依赖 最小结尾 有序二分

🚀 解法演进路径

DP 发现结构从前序状态推出当前答案
→
贪心提炼状态每种长度只保留最小结尾
→
二分加速查找tails 有序，lower_bound 定位

V1 - 动态规划 O(n²)

核心思想：“我能接在谁后面？” 每个位置尝试连接前面所有比它小的、已知的最长递增序列。
  1. dp[i] = 以 nums[i] 结尾的 LIS 长度
  2. 若 nums[j] < nums[i]，则可接在 j 后面
  3. 初始所有 dp[i] = 1
  4. 答案为 max(dp)
dp[i] = max(dp[i], dp[j] + 1)
def lengthOfLIS(nums):
    n = len(nums)
    if n == 0:
        return 0

    dp = [1] * n
    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)

    return max(dp)
时间空间适用
O(n²)O(n)小规模、展示 DP 思维

V2 - 贪心 + 二分 O(n log n)

核心洞察：不关心具体路径，只关心“对于每个长度 L，当前最小结尾是多少”。结尾越小，未来越容易扩展。
  1. 维护 tails：tails[L-1] 表示长度 L 的最小结尾值
  2. 若 x > tails[-1]，直接延长
  3. 否则找第一个 ≥ x 的位置并替换
  4. 答案为 len(tails)
import bisect

def lengthOfLIS(nums):
    tails = []

    for x in nums:
        if not tails or x > tails[-1]:
            tails.append(x)
        else:
            # 找到第一个 >= x 的位置并替换
            i = bisect.bisect_left(tails, x)
            tails[i] = x

    return len(tails)
时间空间优势
O(n log n)O(n)大规模数据最优常用解

✅ 贪心 + 二分为什么正确？

关键性质说明
tails 始终有序新增或替换后仍保持递增结构
替换不影响长度只优化“结尾质量”，不减少已有长度
可用二分定位有序数组支持 lower_bound 快速查找

tails 不一定是真实 LIS，但每个位置都代表某种长度下最有潜力的最小结尾。

判断条件是否满足
是否有“选或不选”的决策？是
当前结果是否依赖前面结果？是
是否存在重叠子问题？是

🎯 方法选择

  • 讲清思路：先写 DP，体现状态定义与转移
  • 追求性能：使用贪心 + 二分，达到 O(n log n)
  • 核心记忆：长度由 tails 数组长度决定，替换是为了降低结尾

🏁 总结一句话

最长递增子序列的本质，是从“状态依赖”出发建立 DP 模型，再通过“贪心维护最小结尾 + 二分加速查找”完成优化。
