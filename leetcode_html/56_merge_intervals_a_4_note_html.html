✓56. 合并区间 - 完整解法总结

1核心问题
将重叠区间合并为不重叠区间的集合
2方法对比：原地 vs 非原地

核心逻辑对比

维度 原地合并法 非原地合并法（推荐）
合并方向向后合并
当前 → 下一个		向前合并
当前 → 结果最后一个
数据修改修改原数组仅修改结果数组
遍历方式while 循环 + 手动索引for 循环自动遍历
代码复杂度较高较低
可读性较差优秀

合并判断条件对比

  • 原地：intervals[i+1][0] ≤ intervals[i][1] —— 下一个区间的起点 ≤ 当前区间的终点
  • 非原地：ans[-1][1] ≥ interval[0] —— 结果最后区间的终点 ≥ 当前区间的起点

合并操作对比

  • 原地：intervals[i+1] = [左, max(右1,右2)] —— 修改原数组，向前覆盖
  • 非原地：ans[-1][1] = max(ans[-1][1], interval[1]) —— 仅更新结果数组的右端点
5贪心策略分析

为什么贪心有效？

  • 1排序保证：左端点有序 → 重叠区间必然相邻
  • 2局部最优：合并所有可合并的相邻区间
  • 3全局最优：得到最大化的不重叠区间集合
合并类型全覆盖
情况1：完全包含　[1,5] + [2,3] → [1,5]
情况2：部分重叠　[1,3] + [2,5] → [1,5]
情况3：相接　　　[1,2] + [2,3] → [1,3]（题目中算重叠）
情况4：无重叠　　[1,2] + [3,4] → 保持两个
7为什么推荐非原地法？
  • 逻辑清晰：能合并就扩展，否则新建
  • 代码简洁：核心逻辑仅 3-4 行代码
  • 不易出错：无需手动管理索引边界检查
  • 工业标准：广泛采用的标准写法
  • 自动边界：for 循环自动处理遍历边界
3核心算法逻辑（推荐）

非原地合并标准解法

# 1. 排序预处理
intervals.sort(key=lambda x: x[0])

# 2. 核心三行逻辑
ans = []
for interval in intervals:
    if ans and interval[0] <= ans[-1][1]:  # 重叠判断
        ans[-1][1] = max(ans[-1][1], interval[1])  # 合并
    else:
        ans.append(interval)  # 新建

关键条件解析

  • ans 非空：避免首次访问 ans[-1]；空结果数组时的保护
  • interval[0] ≤ ans[-1][1]：判断是否重叠；包含等于（相接算重叠）
  • max(ans[-1][1], interval[1])：取最大右端点；覆盖完全包含的情况
4易错点与关键细节

合并条件的等号

✓正确：包含相接情况
if interval[0] <= ans[-1][1]:  # ≤ 包含等于
×错误：会漏掉相接区间
if interval[0] < ans[-1][1]:   # < 排除等于

右端点更新方式

✓正确：取最大值
ans[-1][1] = max(ans[-1][1], interval[1])
×错误：丢失覆盖范围
ans[-1][1] = interval[1]  # 无法处理完全包含的情况
6复杂度分析
时间复杂度
  • 排序：O(n log n) —— 预处理必需（快排/归并）
  • 遍历合并：O(n) —— 一次线性扫描
  • 总计：O(n log n) —— 排序主导
空间复杂度
  • 原地法：O(n) —— 需要结果存储
  • 非原地法：O(n) —— 需要结果存储
  • 排序开销：O(log n) ~ O(n) —— 取决于排序算法
8解题步骤总结

标准流程（四步法）

  1. 1边界检查：空数组直接返回 []
  2. 2排序预处理：intervals.sort(key=lambda x: x[0])
  3. 3遍历合并：使用非原地合并逻辑
  4. 4返回结果：return ans
★

一句话总结

排序定序使重叠相邻，向前合并用结果数组记录；判断用 ≤ 包含相接，更新取 max 确保覆盖完整。非原地法逻辑直观代码简，成为区间合并的标准解法。
