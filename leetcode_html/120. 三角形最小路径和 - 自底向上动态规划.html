三角形最小路径和

🎯 LeetCode 120 自底向上 DP 时间 O(n²) 空间 O(1)

🎯 问题本质 只记规则

从三角形 顶部 走到底部，求路径数字和的 最小值。

移动：每步只能到下一行的 同列 j 或 右邻 j+1。

triangle[i][j] → triangle[i+1][j] / triangle[i+1][j+1]

🔍 核心思想 从底往上

  • 底部节点的答案就是它自己。
  • 上层节点只需选择 下方两个相邻子路径 的较小值。
  • 更新到顶部后，triangle[0][0] 就是全局最优。

🧠 DP 定义 公式

dp[i][j] = 从第 i 行第 j 列到底部的最小路径和
初始：dp[last][j] = triangle[last][j]
转移：dp[i][j] = triangle[i][j] + min(dp[i+1][j], dp[i+1][j+1])
💻 关键代码：原地修改 完整保留｜空间 O(1) 
def minimumTotal(self, triangle: List[List[int]]) -> int:
    # 从倒数第二行开始向上处理
    for row in range(len(triangle) - 2, -1, -1):
        for col in range(len(triangle[row])):
            # 选择下一行相邻两个中较小的路径和
            triangle[row][col] += min(triangle[row+1][col], triangle[row+1][col+1])
    return triangle[0][0]

📊 复杂度 结论

时间O(n²)：每个元素遍历一次。
空间O(1)：复用 triangle 原数组。

⚠️ 易错点 必看

起点从 倒数第二行 开始，不处理最后一行。
循环range(len(triangle)-2, -1, -1)。
子节点只能取 col 和 col+1。
副作用原地修改会改变输入；不想改就复制最后一行做滚动 DP。

🧩 方法对比 推荐

方法空间备注
自顶向下O(n²)/O(n)边界较多
自底向上原地O(1)最推荐
递归记忆化O(n²)思路直观，代码偏长

🧷 速记卡 背这个

方向：从下往上。
选择：当前位置 + min(左下, 右下)。
答案：更新完后返回 triangle[0][0]。
坑：原地 DP 会修改输入。
💡 一句话：从倒数第二行开始，每个位置累加下方相邻较小路径和；最后顶部就是最小路径和，O(n²) 时间，O(1) 空间。
