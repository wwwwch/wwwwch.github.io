136. 只出现一次的数字 - 多解法总结

集合 / 哈希表 / 数学法 / 位运算异或法，对比与适用场景一页速览

📊 方法对比总表

方法 时间复杂度 空间复杂度 关键思路 推荐指数
集合存储（添加/删除） O(n) O(n) 利用集合的唯一性，成对删除 ⭐⭐⭐
哈希表（计数） O(n) O(n) 统计每个数字出现次数 ⭐⭐⭐

🔍 各方法实现详解

1. 集合存储法（添加/删除）

核心思路：利用集合的唯一性，成对出现则抵消，最后剩余即为目标。

def singleNumber(nums):
    seen = set()
    for num in nums:
        if num in seen:
            seen.remove(num)    # 成对出现，从集合中删除
        else:
            seen.add(num)       # 首次出现，加入集合
    return seen.pop()           # 集合中剩余的唯一元素
💡 注意点
  • 需要保证其他数字都出现两次
  • 最后集合中只有一个元素
2. 哈希表（计数法）

核心思路：统计每个数字出现次数，找到次数为1的数字。

def singleNumber(nums):
    count = {}
    for num in nums:
        count[num] = count.get(num, 0) + 1   # 统计频率
    for num, freq in count.items():
        if freq == 1:                        # 找到只出现一次的数字
            return num
💡 注意点
  • 需要遍历两次：一次统计，一次查找
  • 空间复杂度与数字种类数相关
3. 数学方法（集合和与数组和）

核心思路：利用数学公式 2×(a+b+c) - (a+a+b+b+c) = c

def singleNumber(nums):
    return 2 * sum(set(nums)) - sum(nums)
💡 注意点
  • 前提：其他数字都恰好出现两次
  • 在Python中无需担心整数溢出
  • 最简洁但依赖严格的数学条件
4. 位运算（异或法） ✅ 最优解

核心思路：利用异或运算的三个性质：

  1. a ⊕ 0 = a
  2. a ⊕ a = 0
  3. 交换律和结合律：a ⊕ b ⊕ a = b
def singleNumber(nums):
    result = 0
    for num in nums:
        result ^= num   # 依次异或所有数字
    return result
⚙️ 工作原理
假设数组: [a, b, c, a, b] 计算过程: 0 ⊕ a = a a ⊕ b = a⊕b a⊕b ⊕ c = a⊕b⊕c a⊕b⊕c ⊕ a = (a⊕a)⊕b⊕c = 0⊕b⊕c = b⊕c b⊕c ⊕ b = (b⊕b)⊕c = 0⊕c = c ✅

⚠️ 各方法适用条件与限制

方法 适用条件 限制
集合存储 其他数字出现次数为偶数 需要额外 O(n) 空间
哈希表 任何出现次数统计 需要额外 O(n) 空间
数学法 其他数字都出现两次且无溢出风险 仅适用于特定频率
异或法 其他数字出现偶数次 只能处理出现偶数次的情况

💡 关键洞察

🔬 异或运算的巧妙之处
  • 自反性：a ⊕ a = 0 → 成对出现的数字相互抵消
  • 恒等性：a ⊕ 0 = a → 最终留下只出现一次的数字
  • 交换律结合律：运算顺序不影响结果
🎯 为什么异或法是最优解？
优势 说明
时间复杂度 O(n)，只需一次遍历
空间复杂度 O(1)，只使用一个变量
代码简洁性 仅需3行代码
无类型限制 适用于整数、字符等可异或的类型

💎 一句话总结

异或运算巧妙抵消成对数字，O(n) 时间 O(1) 空间最优解；哈希表通用性强但需额外空间；数学法简洁但限制严格。
