11. 盛最多水的容器：双指针

目标：在数组 height 中选两条线，使 面积最大。

双指针 贪心 跳过优化 O(n)

题目核心

  • 输入：[1,8,6,2,5,4,8,3,7]
  • 输出：49
  • 面积由 较短板高度 和 两指针距离 决定。
heightmin(h[p],h[q])
widthq - p
areaheight × width

图像记忆

最大面积 = 7 × 7 = 49

贪心依据

只移动较短的一端，才可能得到更大面积。
移动长板 宽度变小，高度仍受短板限制 → 面积不会变大。
移动短板 宽度变小，但可能遇到更高板 → 面积有机会变大。
易错点：不是移动高度更大的指针；必须移动 较短板。

算法流程

① 初始化p 左端，q 右端
② 计算面积更新最大值
③ 移动短板跳过无效高度
  1. p = 0，q = n - 1，area = 0
  2. while p < q
  3. min(height[p], height[q]) * (q - p)
  4. 移动短板；跳过 <= 当前短板 的位置。

跳过优化

下一个高度 ≤ 当前短板：高度不增、宽度变小 → 面积一定不更大。
  • 可跳过：height[next] <= current_min
  • 不能跳过：可能出现更高板，需要重新计算。

复杂度 & 口诀

时间O(n)
空间O(1)
面积看短板，移动也看短板；短板不变高，宽窄必更小。

完整代码

class Solution:
    def maxArea(self, height: List[int]) -> int:
        # 初始化双指针，p指向左端，q指向右端
        p, q = 0, len(height)-1
        area = 0

        # 当两个指针没有相遇时循环
        while p < q:
            # 计算当前面积：高度取两者较小值 * 宽度
            current_height = min(height[p], height[q])
            width = q - p
            s = current_height * width

            # 更新最大面积
            area = max(area, s)

            # 贪心策略：移动较短的那一端
            # 因为宽度在减小，如果高度不增加，面积一定更小，无需计算
            if height[p] <= height[q]:
                current_min = height[p]
                p += 1
                # 优化：快速跳过比当前短板更短或相等的线
                while p < q and height[p] <= current_min:
                    p += 1
            else:
                current_min = height[q]
                q -= 1
                # 优化：快速跳过比当前短板更短或相等的线
                while p < q and height[q] <= current_min:
                    q -= 1

        return area
