LeetCode 11. 盛最多水的容器

目标：在非负整数数组 height 中，找出两条线与 x 轴围成面积最大的容器。核心问题是如何高效搜索最大面积。

V1 双指针法：贪心移动短板

核心思想

使用双指针从两端向内收缩，每一步根据“短板效应”决定移动哪一侧指针。

  • 左指针：p = 0，右指针：q = n - 1
  • 当前宽度：q - p
  • 有效高度：min(height[p], height[q])
area = (q - p) × min(height[p], height[q])

关键观察：为什么移动短板？

只有移动较短的那一端，才有可能得到更大的面积。
移动长板：不会变大 宽度减小，高度仍受短板限制，面积不可能增加。
移动短板：可能更优 宽度虽减小，但可能遇到更高的板，从而提升有效高度。

图形记忆

蓝/红柱：当前左右边界 浅蓝区：当前容器面积

算法流程

① 初始化
p = 0
q = len(height)-1
max_area = 0
② 计算面积
area = (q-p) × min(...)
更新 max_area
③ 移动短板
height[p] < height[q] → p += 1
否则 q -= 1
④ 返回答案
循环直到 p >= q
返回 max_area

V1 代码

def max_area(height):
    p, q = 0, len(height) - 1
    max_area = 0

    while p < q:
        h = min(height[p], height[q])
        max_area = max(max_area, (q - p) * h)

        if height[p] < height[q]:
            p += 1
        else:
            q -= 1

    return max_area

复杂度：时间复杂度 O(N)，空间复杂度 O(1)。

V1.1 跳过无效状态

优化思路

移动指针时，如果下一个位置的高度 ≤ 当前短板，则新容器的高度不会增加，宽度还更小，面积一定不会更大。

因此可以跳过这些无意义的位置。

while p < q and height[p] <= h:
    p += 1

适用场景

  • 移动指针后，连续多个柱子都很矮
  • 有大量重复或递减数据
  • 减少不必要面积计算，常数级性能提升

最坏复杂度仍为：O(N)

V1.1 优化代码

def max_area_optimized(height):
    p, q = 0, len(height) - 1
    max_area = 0

    while p < q:
        h = min(height[p], height[q])
        max_area = max(max_area, (q - p) * h)

        if height[p] < height[q]:
            while p < q and height[p] <= h:
                p += 1
        else:
            while p < q and height[q] <= h:
                q -= 1

    return max_area

正确性证明（简要）

假设最优解由 (i, j) 构成，且 i < j。双指针从两端向内收缩，一定会在某一时刻先到达其中一个边界。

由于策略始终是保留长板、移动短板，不会因为移动长板而浪费可能性，也不会提前跳过最优解的两个端点，因此最终能覆盖到最大面积。

一句话记忆

使用双指针从外向内收缩，基于“移动短板才可能增大面积”的贪心策略，在 O(N) 时间内找到最大盛水面积。

进一步优化：移动短板时，跳过所有 <= 当前短板高度 的位置，避免无效计算。

记忆核心：面积 = 宽度 × 短板高度；宽度每次都会变小，因此只有寻找更高的短板，才有机会让面积变大。
