128. 最长连续序列｜O(n) 复习速记

目标：未排序数组 nums 中，找最长“连续整数链”的长度。

HashSet只从起点扩展去重防退化O(n)

核心图解

set(nums)去重 + O(1) 查询

向右扩展x+1,x+2...

输入[100,4,200,1,3,2]
最长链1→2→3→4
答案4

记忆句：前面没人，才开跑；前面有人，直接跳。

必须保留的结论

  1. 不能排序：排序是 O(nlogn)，不满足要求。
  2. 必须用 HashSet：成员查询平均 O(1)。
  3. 只从起点算：x-1 not in st 才扩展。
  4. 长度公式：扩展到第一个不存在的 y，长度为 y-x。
①建 set
②判起点
③向右扫
④更新 ans

易错点：遍历谁？

错：遍历 nums

重复元素会反复触发内层循环，最坏可能退化为 O(n²)。

对：遍历 st

每个不同数字只作为候选检查一次，总体保持 O(n)。

反例记忆：[1,1,1,...,1,2,3,4,5,...]

复杂度 & 提前结束

  • 时间：O(n)
  • 空间：O(m)，m=len(set(nums))
  • 小优化：若 2*ans >= m，后面不可能更长，提前 break。

一眼回忆版

set 去重 → x-1 不存在才是起点 → while 找 x+1 → 长度 y-x → 取最大。

核心防线：不排序；不重复扫；不从中间开始。

完整代码｜不可删改逻辑与注释Python
class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        st = set(nums)  # 把 nums 转成哈希集合
        m = len(st)

        ans = 0
        for x in st:  # 遍历哈希集合
            if x-1 in st:  # 如果 x 不是序列的起点，直接跳过
                continue

            # x 是序列的起点
            y = x+1
            while y in st:  # 不断查找下一个数是否在哈希集合中
                y += 1

            # 循环结束后，y-1 是最后一个在哈希集合中的数
            ans = max(ans, y-x)  # 从 x 到 y-1 一共 y-x 个数

            # 小优化
            if 2*ans >= m:  # ans 不可能变得更大
                break

        return ans
