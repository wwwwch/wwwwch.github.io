01一、NLP大模型 / CV大模型

NLP大模型、CV大模型

NLP大模型主要建模语言序列中的语义关系，CV大模型主要建模图像中的空间结构和视觉模式。

关键要点
  • 数据类型：NLP 处理文本 token 序列；CV 处理图像/视频像素。
  • 核心任务：NLP 用于对话、翻译、摘要；CV 用于分类、检测、分割。
  • 模型结构：NLP 现在基本以 Transformer 为主；CV 早期以 CNN 为主，后来也大量使用 Transformer（如 ViT）。
  • 发展趋势：两者逐渐融合，演进为多模态大模型。
我的理解

本质区别是序列建模 vs 空间建模，但底层方法正在统一，尤其是在 Transformer 和 Attention 出现以后，文本和图像都可以转成 token 来处理。

问NLP和CV大模型的根本区别是什么？
答根本区别在于输入结构不同。NLP 更关注上下文语义依赖，CV 更关注局部纹理和空间关系。但从发展趋势看，两者正在被统一到同一种建模框架里。

02二、Transformer / Attention

Transformer 的注意力机制

Transformer 通过自注意力机制（Attention）实现序列中全局信息交互，是其核心创新。

关键要点
  • 核心思想：每个 token 都与其他 token 计算相关性，然后加权汇总信息。
  • 优势：能捕捉长距离依赖，不像 RNN 那样必须按顺序传递。
  • 并行能力：可并行处理整个序列，训练效率高。
  • 多头注意力：让模型从不同子空间学习不同关系。
我的理解

可以把它理解成：每个词在编码时，都会“看一遍”句子里其他所有词，判断哪些信息更重要，再决定如何更新自己的表示。

问为什么 Transformer 比 RNN 更好？
答RNN 是顺序传递信息，长距离依赖容易丢失，而且不能高效并行；Transformer 用 Attention 直接建立全局依赖，既提升了效果，也提升了训练效率。
问多头注意力有什么意义？
答多头注意力相当于让模型从多个视角同时建模，比如有的头关注语法关系，有的头关注语义相关性，能提升表达能力。

Transformer 的位置编码、QKV 实现计算

Transformer 通过自注意力机制（Attention）+ 位置编码（Positional Encoding）+ QKV计算来建模序列关系。

关键要点
  • 注意力机制（Self-Attention）：每个 token 与其他 token 计算相关性，加权汇总信息，捕捉全局依赖。
  • 公式：Attention(Q,K,V)=softmax((QKᵀ)/√dₖ)V。
  • QKV计算：输入向量经过三组线性变换，得到 Q（Query）、K（Key）、V（Value）。Q 表示“我要找什么”，K 表示“我有什么”，V 表示“我的内容是什么”。
  • 位置编码（Positional Encoding）：由于 Attention 本身不包含顺序信息，所以需要显式加入位置信息，常见方式是正弦余弦编码或可学习位置编码。
  • 整体作用：Attention 负责“信息交互”，QKV 是具体实现方式，位置编码补充“顺序信息”。
我的理解

QKV 本质上是在做一种“查询-匹配-取值”机制，位置编码则是告诉模型“词的内容之外，它还处在句子的什么位置”。

问为什么 Attention 里要除以 √dk？
答因为 Q 和 K 点积后数值可能很大，会让 softmax 过于尖锐，导致梯度不稳定。除以 √dk 是为了做缩放，保证训练更稳定。
问位置编码为什么重要？
答因为自注意力本身不区分先后顺序，如果没有位置编码，模型只知道“有哪些词”，不知道“谁在前谁在后”。

03三、经典深度学习模型

ResNet 及跳跃连接

ResNet 通过残差连接（skip connection）解决深层网络训练中的退化问题。

关键要点
  • 残差思想：网络不直接学习目标映射，而是学习残差 F(x)，输出为 F(x)+x。
  • 跳跃连接作用：保留原始输入信息，缓解梯度消失。
  • 优势：使网络能训练得更深，同时保持稳定。
  • 应用：图像分类、检测、分割中都很常见。
我的理解

ResNet 不是让网络“从零学一个完整映射”，而是让它在原输入基础上学“修正量”，这样更容易优化。

问ResNet 为什么能训练更深？
答因为跳跃连接让梯度可以直接回传，同时保留了原始特征信息，网络只需要学习增量部分，所以更容易优化。
问跳跃连接和 U-Net 里的跳跃连接一样吗？
答不完全一样。ResNet 的跳跃连接主要是为了优化训练、缓解梯度问题；U-Net 的跳跃连接更多是为了融合浅层细节和深层语义。

U-Net

U-Net 是一种典型的编码器-解码器结构的图像分割模型，特别适合像素级预测任务。

关键要点
  • 结构特点：左侧编码器负责提取语义特征，右侧解码器负责恢复分辨率。
  • 跳跃连接：将浅层细节特征与深层语义特征融合。
  • 核心优势：既能理解“图里是什么”，又能精确定位“它在哪”。
  • 应用场景：医学图像分割、遥感分割等。
我的理解

U-Net 的价值在于把“高层语义”和“低层边界细节”结合起来，这正是分割任务最需要的能力。

问U-Net 为什么适合分割？
答因为分割不仅要知道目标是什么，还要知道每个像素属于哪里。U-Net 同时保留了语义信息和空间细节，所以效果好。

YOLO

YOLO（You Only Look Once）是一种单阶段目标检测模型，特点是速度快、适合实时场景。

关键要点
  • 核心思想：把目标检测统一成一个回归问题，同时预测边框和类别。
  • 单阶段特点：不需要先生成候选框，再分类。
  • 优势：端到端、推理速度快。
  • 典型应用：实时监控、自动驾驶、工业检测。
我的理解

YOLO 的核心价值在于“快”，适合对时延敏感的场景。它不是一步一步找目标，而是一次前向传播直接把目标框和类别都预测出来。

问YOLO 为什么快？
答因为它是单阶段方法，一次前向计算就完成检测，不像两阶段方法那样还要生成候选区域再分类。
问YOLO 和 Faster R-CNN 的区别？
答YOLO 更快，适合实时；Faster R-CNN 通常精度更高，但推理较慢。核心区别是单阶段 vs 两阶段。

LSTM

LSTM 是一种改进的 RNN，专门为了解决长序列建模中的梯度消失问题。

关键要点
  • 核心结构：输入门、遗忘门、输出门。
  • 作用：控制信息的写入、保留和输出。
  • 优势：相比普通 RNN，更能保留长期依赖信息。
  • 应用：时间序列预测、语音识别、早期 NLP 任务。
我的理解

LSTM 的本质是给网络加了一个“记忆管理系统”，让它知道哪些信息该记住，哪些信息该忘记。

问LSTM 解决了 RNN 的什么问题？
答主要解决梯度消失问题。普通 RNN 在长序列上很难保留早期信息，LSTM 通过门控机制增强了长期记忆能力。
问LSTM 和 Transformer 的区别？
答LSTM 是递归式顺序建模，擅长时间依赖但难并行；Transformer 是全局并行建模，效果和效率通常更好，所以现在主流 NLP 多采用 Transformer。

04四、传统机器学习

XGBoost

XGBoost 是一种基于梯度提升树（Gradient Boosting Tree）的集成学习算法，在结构化数据上表现非常强。

关键要点
  • 核心思想：一棵树一棵树地学，每一轮拟合前一轮的残差。
  • 优化能力：加入正则化，支持二阶导优化，训练更稳定。
  • 优点：精度高、鲁棒性强、可解释性较好。
  • 适用场景：风控、推荐、用户画像、表格数据建模。
我的理解

如果说深度学习擅长非结构化数据，那 XGBoost 可以说是表格数据领域的“强基线”。

问XGBoost 为什么效果好？
答因为它结合了 boosting 的逐步纠错思想、正则化控制复杂度以及高效工程实现，所以在很多结构化数据任务里表现非常稳定。
问XGBoost 和随机森林的区别？
答随机森林是并行建多棵树再投票，强调“降低方差”；XGBoost 是串行建树、逐步拟合误差，强调“降低偏差”，通常精度更高。

决策树原理、如何用于回归

决策树通过不断选择特征切分数据空间，最终形成树状结构进行预测。用于回归时，本质是做分段函数拟合。

关键要点
  • 基本原理：每次选择一个特征和阈值，把数据划分成更纯的子集。
  • 分类任务：通常用信息增益、基尼指数等指标。
  • 回归任务：通常以最小化均方误差（MSE）为目标。
  • 回归输出：叶子节点通常输出该区域样本的均值。
我的理解

决策树本质是在特征空间里不断“切块”，分类时输出类别，回归时输出一个连续值。

问决策树为什么能做回归？
答因为它并不要求输出一定是类别，只要把样本划分到某个叶子节点，就可以用该节点样本的均值来表示预测值。
问决策树有哪些缺点？
答容易过拟合，对数据扰动敏感，单棵树泛化能力有限，所以实际中常与集成方法结合使用。

无监督学习

无监督学习是在没有标签的数据上发现数据内在结构或规律的方法。

关键要点
  • 典型任务：聚类、降维、密度估计、异常检测。
  • 常见方法：K-Means、PCA、自编码器。
  • 作用：做数据探索、特征学习、数据预处理。
  • 特点：没有人工标签，依赖数据本身结构。
我的理解

它更像是在问：“这些数据自己能不能分出规律？”而不是“给定标签后怎么预测”。

问无监督学习的核心价值是什么？
答核心价值是从无标签数据中提取结构信息，特别适合真实场景中标签昂贵或难获取的任务。
问PCA 的本质是什么？
答PCA 是寻找数据方差最大的方向，把高维数据投影到更低维空间，本质是一种线性降维方法。

05五、大模型与API工程

GPT API调用、有哪些问题、未来发展前景

GPT API 调用本质上是把大模型能力通过接口方式集成到具体业务系统中。

关键要点
  • 调用方式：输入 prompt 或消息列表，请求模型生成回复。
  • 常见问题：幻觉——可能生成看似合理但不准确的内容；成本——高频调用会带来较高 token 成本；延迟——模型越大、上下文越长，响应越慢；上下文限制——模型虽然可处理长上下文，但并不等于记忆稳定。
  • 常见优化方式：Prompt工程、RAG、函数调用/工具调用、缓存。
  • 未来趋势：从“单轮问答”走向“Agent执行”；从“纯文本”走向“多模态交互”；从“云端依赖”走向“低成本、小型化、本地部署”。
我的理解

GPT API 已经不是单纯聊天接口，而是在向“通用智能能力接口”发展。未来重点不只是模型更强，而是更稳定、更便宜、更容易接入业务。

问GPT API 最大的问题是什么？
答我认为主要是三点：准确性、成本和可控性。模型很强，但并不天然可靠，所以业务中往往需要检索增强、输出约束和人工兜底。
问RAG 是什么，解决什么问题？
答RAG 是检索增强生成，即先从外部知识库检索相关内容，再把结果提供给模型生成答案。主要解决知识过时和幻觉问题。
问为什么大模型还需要工具调用？
答因为模型擅长生成和推理，但不擅长直接访问外部世界。工具调用让模型能查数据、算结果、执行操作，能力会更完整。

06六、大模型部署 / 微调

大模型部署、微调

大模型部署是把模型真正落地到可用环境中，微调是让模型更贴合具体场景和任务。

关键要点
  • 部署目标：让模型在云端或本地稳定提供推理服务。
  • 部署方式：云端部署——统一管理、扩展方便；本地部署——适合隐私敏感、低延迟场景。
  • 常见挑战：显存占用大、推理延迟高、并发压力大。
  • 优化手段：量化、蒸馏、KV Cache、推理加速框架。
  • 微调方式：全量微调——更新全部参数，效果强但成本高；参数高效微调——如 LoRA、PEFT，只训练少量参数。
  • 微调价值：让模型更适应行业术语、风格、任务逻辑。
我的理解

部署解决的是“模型能不能跑得起来、扛得住业务压力”，微调解决的是“模型是不是更懂我的场景”。

问Prompt工程和微调怎么选？
答如果需求偏轻、变化快、预算有限，优先 Prompt；如果任务固定、数据明确、对稳定性要求高，可以考虑微调。
问LoRA 为什么高效？
答因为它不直接更新整个大模型参数，而是只在某些层上增加低秩矩阵，训练参数量和显存占用都更低。
问大模型部署的难点有哪些？
答主要是算力成本、推理时延、并发吞吐、稳定性和安全控制。模型能力强，但工程代价也很高。

07七、整体串联理解

如何把这些知识点串起来理解？

可以把这些知识点分成三层：基础模型层、统一建模层、应用工程层。

关键要点
  • 基础模型层：包括决策树、XGBoost、LSTM、ResNet、U-Net、YOLO。这一层主要解决具体任务，比如分类、回归、检测、分割、序列建模。
  • 统一建模层：Transformer 和 Attention 是核心。它们把文本、图像等不同模态逐步统一到一个共同框架中。
  • 应用工程层：包括 GPT API、RAG、部署、微调。这一层重点不是“模型原理”，而是“怎么把模型真正用起来，并稳定落地业务”。
我的理解

面试里如果能体现这种分层认知，通常会比单独背概念更有说服力。因为这说明你不是在记术语，而是知道它们在整个 AI 技术体系里的位置。

08八、面试高频总结版（适合结尾）

AI 技术发展路线总结回答

如果面试官问“你怎么理解现在 AI 技术的发展路线”，可以用一段从专用模型到完整系统的回答。

关键要点
  • 总结回答：我理解现在 AI 技术的发展，大致是从“针对单任务的专用模型”，走向“基于 Transformer 的统一大模型”，再走向“结合检索、工具调用和工程部署的完整系统”。
  • 也就是说，前几年更关注模型结构本身，现在越来越关注模型如何落地，包括多模态、Agent、低成本部署和业务稳定性。
