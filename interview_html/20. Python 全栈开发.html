Python 全栈开发常用技术栈复习笔记

核心理解：Python 全栈开发不是只会 Python 后端，而是要理解从前端页面、接口通信、后端框架、业务模型、数据库、中间件、部署运维到测试安全的完整链路。

用户浏览器
  ↓
前端：React / Vue / HTML / CSS / JavaScript / TypeScript
  ↓
前端构建：Vite / Webpack / npm / pnpm
  ↓
网络通信：HTTP / REST API / JSON / WebSocket
  ↓
Python 后端：FastAPI / Flask / Django
  ↓
业务与数据模型：Pydantic / ORM / 认证 / 权限 / 任务调度
  ↓
数据与中间件：PostgreSQL / MySQL / MongoDB / Redis / RabbitMQ / Kafka
  ↓
部署运维：Linux / Nginx / Uvicorn / Gunicorn / Docker / CI/CD
  ↓
质量保障：测试 / 日志 / 监控 / 安全 / 性能优化

一、前端基础技术

前端负责用户看到的页面、交互行为和数据展示。

技术 作用 特点与场景
HTML 定义页面结构 标题、表单、按钮、列表、表格等页面骨架
CSS 控制页面样式 颜色、布局、字体、响应式适配
JavaScript 实现页面交互 点击事件、表单校验、调用后端接口
TypeScript 带类型系统的 JavaScript 适合中大型前端项目，减少类型错误，提高可维护性
React 前端 UI 框架 生态强、灵活度高，适合复杂交互应用
Vue 前端 UI 框架 上手相对简单，常用于后台管理系统、中小型业务系统

简单示例：HTML + JavaScript 调用后端 API

<input id="task" placeholder="请输入任务" />
<button onclick="createTask()">创建任务</button>

<script>
async function createTask() {
  const title = document.getElementById("task").value;

  await fetch("/api/tasks", {
    method: "POST",
    headers: {"Content-Type": "application/json"},
    body: JSON.stringify({ title })
  });
}
</script>

面试记忆：HTML 是结构，CSS 是样式，JavaScript 是交互，TypeScript 是更工程化的 JavaScript，React/Vue 是用来组织复杂页面的前端框架。

二、前端构建工具

前端构建工具负责依赖管理、开发服务器、代码打包、资源压缩和生产环境构建。

技术 作用 区别与场景
npm Node.js 默认包管理器 安装 React、Vue、Axios 等前端依赖
pnpm 现代包管理器 安装速度快，节省磁盘空间，现代项目常用
Webpack 老牌前端打包工具 功能强大，配置复杂，很多老项目仍在使用
Vite 现代前端构建工具 启动快，配置简单，React/Vue 新项目常用

简单命令

# 安装依赖
pnpm install

# 启动开发环境
pnpm dev

# 构建生产环境文件
pnpm build

面试记忆：npm/pnpm 负责管理依赖，Vite/Webpack 负责编译和打包。新项目一般更常见 Vite，老项目可能使用 Webpack。

三、网络通信层

网络通信层负责前端和后端之间的数据交换。

技术 作用 场景
HTTP 浏览器和服务端通信的基础协议 网页访问、接口调用
REST API 一种常见 API 设计风格 使用 GET、POST、PUT、DELETE 表示查询、新增、修改、删除
JSON 前后端传输数据的常用格式 轻量、易读、前后端都支持
WebSocket 浏览器和服务端之间的双向实时通信 聊天、实时通知、行情推送、协同编辑

REST API 示例

GET    /api/tasks        查询任务列表
POST   /api/tasks        新建任务
GET    /api/tasks/1      查询单个任务
PUT    /api/tasks/1      更新任务
DELETE /api/tasks/1      删除任务

JSON 示例

{
  "id": 1,
  "title": "完成周报",
  "done": false
}

面试记忆：普通业务系统主要使用 HTTP + REST API + JSON；如果需要实时双向通信，再考虑 WebSocket。

四、Python 后端框架

后端负责接收请求、执行业务逻辑、访问数据库、返回响应结果。

框架 特点 适合场景
FastAPI 现代、高性能、类型注解友好、自动生成接口文档 REST API、微服务、AI 服务、前后端分离项目
Flask 轻量、灵活、简单，扩展由开发者自由选择 小项目、原型系统、轻量接口服务
Django 大而全，自带 ORM、Admin、用户系统、模板系统 后台管理系统、CMS、传统 Web 应用、业务系统

FastAPI 简单示例

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class TaskCreate(BaseModel):
    title: str

@app.post("/api/tasks")
def create_task(task: TaskCreate):
    return {
        "id": 1,
        "title": task.title,
        "done": False
    }

说明：FastAPI 会自动解析 JSON 请求体，并用 Pydantic 校验数据结构。

Flask 简单示例

from flask import Flask, request, jsonify

app = Flask(__name__)

@app.route("/api/tasks", methods=["POST"])
def create_task():
    data = request.json
    return jsonify({
        "id": 1,
        "title": data["title"],
        "done": False
    })

说明：Flask 更轻量，很多功能需要自己选择扩展库。

Django 简单理解

# Django 通常会包含：
# models.py    定义数据库模型
# views.py     处理请求
# urls.py      配置路由
# admin.py     配置后台管理

说明：Django 更像“全家桶”，适合快速开发完整业务系统。

后端选型总结：

  • 只写 API、AI 服务、微服务：优先考虑 FastAPI
  • 项目小、想灵活控制：可以用 Flask
  • 需要后台管理、用户系统、ORM 一体化：适合 Django

五、业务与数据模型层

这一层是后端开发的重点，负责数据校验、业务规则、数据库模型、登录认证、权限控制和后台任务。

1. Pydantic

Pydantic 是 Python 的数据校验和序列化库。在 FastAPI 中常用于定义请求体、响应体和配置模型。

能力 说明
数据校验 判断字段是否符合类型和约束
类型转换 例如把字符串 "123" 转成整数 123
嵌套模型 支持复杂 JSON 数据结构
序列化 把 Python 对象转成 dict 或 JSON 
from pydantic import BaseModel, Field

class UserCreate(BaseModel):
    username: str = Field(min_length=3, max_length=20)
    age: int = Field(ge=0, le=120)

面试重点：Pydantic 不是静态类型检查器，而是运行时数据校验工具。

2. ORM

ORM，全称 Object Relational Mapping，对象关系映射。它允许开发者用 Python 对象操作数据库，而不是直接写 SQL。

ORM 特点 场景
SQLAlchemy Python 生态常用 ORM，灵活强大 FastAPI、Flask 项目常用
Django ORM Django 自带 ORM，和框架集成度高 Django 项目默认使用 
# ORM 思路示例
user = User(name="Alice", age=20)
db.add(user)
db.commit()

等价于执行类似 SQL：

INSERT INTO users (name, age) VALUES ('Alice', 20);

面试重点：ORM 提高开发效率，但复杂查询、性能优化、索引设计仍需要理解 SQL。

3. 认证 Authentication

认证解决“你是谁”的问题。

常见方式：

  • Session 登录
  • Cookie 登录
  • Token 登录
  • JWT
  • OAuth2
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...

典型流程：

用户登录
  ↓
服务端校验用户名密码
  ↓
返回 Token / Session
  ↓
前端后续请求携带 Token
  ↓
后端识别当前用户

4. 权限 Authorization

权限解决“你能做什么”的问题。

概念 含义
认证 确认用户身份，例如你是谁
权限 判断用户是否有资格执行某个操作 
普通用户：只能查看自己的任务
管理员：可以查看所有用户任务

常见权限模型：

  • RBAC：基于角色的权限控制，例如管理员、运营、普通用户
  • ABAC：基于属性的权限控制，例如部门、地区、资源归属

5. 任务调度与异步任务

有些任务不适合在接口请求中同步完成，例如发邮件、生成报表、处理图片、调用慢接口。

技术 作用 场景
Celery Python 常用异步任务框架 发邮件、报表生成、后台任务
APScheduler Python 定时任务库 定时清理数据、定时报表 
用户提交订单
  ↓
接口立即返回成功
  ↓
后台任务异步发送邮件、通知、生成报表

六、数据与中间件

1. 关系型数据库：PostgreSQL / MySQL

关系型数据库用表、行、列组织数据，适合结构清晰、需要事务一致性的业务。

数据库 特点 场景
PostgreSQL 功能强，支持复杂查询、事务、JSON、扩展能力好 复杂业务系统、数据一致性要求高的项目
MySQL 使用广泛，生态成熟，性能稳定 电商、后台系统、常规业务系统
SQLite 轻量、无需单独数据库服务 本地开发、小工具、Demo 项目 
SELECT * FROM tasks WHERE user_id = 1;

2. MongoDB

MongoDB 是文档型数据库。它以类似 JSON 的文档形式存储数据，适合结构变化较频繁的数据。

{
  "title": "完成周报",
  "tags": ["工作", "周报"],
  "metadata": {
    "priority": "high"
  }
}

场景：内容系统、日志、配置、非固定结构数据。

3. Redis

Redis 是内存型数据存储，常用于缓存、计数器、限流、分布式锁、排行榜等。

用途 例子
缓存 缓存热点商品详情，减少数据库压力
验证码 保存短信验证码，设置过期时间
限流 限制同一 IP 每分钟访问次数
排行榜 使用有序集合维护分数排行
任务队列 Broker Celery 可以使用 Redis 作为消息中间件 
# 逻辑示例
先查 Redis 缓存
如果没有，再查数据库
查到后写入 Redis
最后返回给前端

4. RabbitMQ 和 Kafka

二者都属于消息系统，但定位不同。

技术 特点 适合场景
RabbitMQ 传统消息队列，强调可靠投递和任务分发 订单处理、邮件发送、异步任务
Kafka 高吞吐分布式日志系统，适合流式数据 日志采集、用户行为埋点、实时数据流 
RabbitMQ：更像任务队列
Kafka：更像高吞吐数据管道

七、部署与运维

部署运维层负责把项目稳定运行在服务器或云环境中。

技术 作用 场景
Linux 服务器操作系统 大多数后端服务部署在 Linux 上
Nginx 反向代理、静态资源服务、HTTPS 入口 把外部请求转发给 Python 后端
Uvicorn ASGI Server 常用于运行 FastAPI
Gunicorn Python 进程管理服务器 常用于 Flask、Django，也可配合 Uvicorn Worker
Docker 容器化部署 统一开发、测试、生产环境
CI/CD 自动化测试、构建和发布 代码提交后自动执行测试和部署流程

典型部署链路

用户浏览器
  ↓
Nginx
  ↓
Uvicorn / Gunicorn
  ↓
FastAPI / Flask / Django
  ↓
PostgreSQL / MySQL / Redis

Docker Compose 简单理解

一个项目可能同时启动：
- backend 后端服务
- frontend 前端服务
- postgres 数据库
- redis 缓存
- nginx 网关

面试记忆：Nginx 是入口，Uvicorn/Gunicorn 是运行 Python Web 应用的服务，Docker 用来封装环境，CI/CD 用来自动发布。

八、质量保障

1. 测试

测试用于验证代码是否符合预期。

类型 说明
单元测试 测试单个函数或模块
接口测试 测试 API 请求和响应是否正确
集成测试 测试多个模块组合后的行为 
def test_add():
    assert 1 + 1 == 2

Python 常用测试框架：pytest。

2. 日志

日志用于记录系统运行过程，方便排查问题和做审计。

import logging

logging.info("user created task")
logging.error("database connection failed")

常见日志内容：

  • 请求路径
  • 用户 ID
  • 错误堆栈
  • 接口耗时
  • 关键业务操作

3. 监控

监控用于观察系统健康状态。

监控对象 例子
系统资源 CPU、内存、磁盘、网络
接口状态 QPS、响应时间、错误率
业务指标 注册数、订单数、支付成功率

常见工具：Prometheus、Grafana、Sentry。

4. 安全

安全点 说明
HTTPS 加密传输，防止数据被窃听
密码加密 密码不能明文存储，应使用哈希算法
JWT / Session 用于用户登录态管理
CORS 控制跨域访问
SQL 注入防护 使用参数化查询或 ORM，不拼接 SQL
XSS 防护 避免用户输入被当作脚本执行
CSRF 防护 防止用户在不知情情况下提交恶意请求

安全重点：认证不等于权限。认证是识别用户身份，权限是判断用户是否能执行某个操作。

5. 性能优化

方向 常见做法
数据库优化 建立索引、分页查询、避免 N+1 查询、优化慢 SQL
缓存优化 使用 Redis 缓存热点数据
接口优化 减少不必要字段、压缩响应、异步处理耗时任务
并发优化 合理使用异步 I/O、任务队列、连接池
前端优化 代码分包、懒加载、静态资源缓存

九、一个完整业务例子：任务管理系统

假设开发一个简单的任务管理系统，用户可以登录、创建任务、查看任务列表。

层级 涉及技术 作用
前端 React / Vue / HTML / CSS / TypeScript 展示登录页、任务列表页、创建任务表单
前端构建 Vite / pnpm 启动前端开发环境，打包生产静态文件
网络通信 HTTP / REST API / JSON 前端通过接口和后端交换数据
后端 FastAPI 提供登录、创建任务、查询任务接口
数据模型 Pydantic / ORM 校验请求数据，操作数据库中的用户表和任务表
认证权限 JWT / RBAC 判断用户是否登录，是否只能访问自己的任务
数据库 PostgreSQL / MySQL 保存用户和任务数据
缓存 Redis 缓存用户信息、验证码、热点数据
异步任务 Celery / RabbitMQ 异步发送任务提醒邮件
部署 Nginx / Uvicorn / Docker / Linux 把系统部署到服务器
质量保障 pytest / logging / Prometheus 测试接口、记录日志、监控服务状态

请求链路示例

用户点击“创建任务”
  ↓
React / Vue 收集表单数据
  ↓
fetch / axios 发送 POST /api/tasks
  ↓
FastAPI 接收请求
  ↓
Pydantic 校验 title 是否为空
  ↓
认证模块识别当前用户
  ↓
权限模块判断用户是否能创建任务
  ↓
ORM 写入 PostgreSQL / MySQL
  ↓
Redis 可缓存任务列表
  ↓
消息队列异步发送提醒邮件
  ↓
后端返回 JSON
  ↓
前端刷新任务列表

2. FastAPI、Flask、Django 怎么选？

框架 一句话区别
FastAPI 适合现代 API 服务，类型提示强，自动文档友好
Flask 轻量灵活，适合小项目和高度定制
Django 全家桶，适合完整业务系统和后台管理

3. Pydantic 和 ORM 的区别？

技术 作用
Pydantic 负责数据校验、类型转换、请求响应模型
ORM 负责把 Python 对象映射到数据库表，操作数据库

4. Redis、RabbitMQ、Kafka 的区别？

技术 定位
Redis 高速缓存和内存数据结构存储
RabbitMQ 可靠消息队列，适合任务分发
Kafka 高吞吐消息流，适合日志和实时数据管道

5. 认证和权限的区别？

认证 Authentication：确认你是谁
权限 Authorization：确认你能做什么

6. Nginx、Uvicorn、Gunicorn 的区别？

技术 作用
Nginx 对外入口，负责反向代理、HTTPS、静态资源
Uvicorn ASGI Server，常用于 FastAPI
Gunicorn Python 进程管理服务，常用于 Flask/Django，也可管理 Uvicorn Worker

十一、最终记忆版

前端：
HTML/CSS/JS/TS + React/Vue，负责页面和交互。

构建：
npm/pnpm 管依赖，Vite/Webpack 负责开发和打包。

通信：
HTTP + REST API + JSON 是主流，WebSocket 用于实时通信。
